# Architecture Design Document — Template

Write the document as `architecture-design-document.md` in the output folder,
using exactly the Markdown subset that `scripts/build_docx.py` supports
(headings `#`–`####`, paragraphs with `**bold**`/`*italic*`/`` `code` ``,
`-`/`1.` lists, `|` tables, `![caption](path)` images, ``` code blocks,
`>` callouts). Image paths are relative to the `.md` file, e.g.
`diagrams/01-system-context.png`.

Follow the section skeleton below. Text in [square brackets] is instructions
to you — replace it with real content, never copy it into the document.
Skip a subsection only when the repository genuinely has nothing for it, and
say so in one line ("No caching layer was found in the codebase.") rather
than deleting the heading silently. Every factual claim must trace back to
something you read during the scan — name files, projects, classes and
packages. Never invent components, patterns, or infrastructure that you did
not see evidence for.

Writing style: professional, factual, present tense. Prefer "The API project
registers `ShopContext` with a SQL Server provider (`Program.cs`)" over vague
claims like "the system uses a modern layered architecture".

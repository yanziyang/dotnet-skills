# Module Design Specification — Template

Write the document as `module-design-specification.md` using exactly the
Markdown subset `scripts/build_docx.py` supports: headings `#`–`####`,
paragraphs with `**bold**`/`*italic*`/`` `code` ``, `-`/`1.` lists, `|`
tables, `![caption](path)` images, ``` code blocks, `>` callouts. Image
paths are relative to the `.md` file (`diagrams/05-module-structure.png`).

Text in [square brackets] is instructions to you — replace with real
content. The MDS is the developer-facing counterpart of the FS: it names
real projects, classes, methods, tables and routes, and it references FS
requirement IDs (`FR-…`) so the two documents cross-link. Cite evidence
(file paths, class names) for every design claim; never invent structure.

A "module" is normally a project in the solution. If one project clearly
contains several independent areas (e.g. vertical slices), treat each area
as a module and say how you partitioned it.

# Documentation Templates

Exact skeletons for every file in `Documentation/`. Follow the headings
verbatim — they are the merge contract for future runs. Text in `<angle
brackets>` is a placeholder to replace; everything else is literal. Filled
example rows show the expected density: short, factual, path-cited.

Every file starts with the same two lines under its H1:

```markdown
# <Title>

> Last updated: 2026-07-09 · commit a1b2c3d · generated by maintain-knowledge
```

Size budgets (hard limits — cut detail, not accuracy, to meet them):
`KnowledgeBase.md` ≤ 150 lines · `README.md` ≤ 60 · every other file ≤ 400.

---

## KnowledgeBase.md (agent-optimised — ≤150 lines)

Dense facts for an LLM agent starting a task in this repo. No prose
paragraphs, no marketing, no repetition of other docs — just the facts an
agent needs before touching code:

```markdown
# Knowledge Base

> stamp line

## Facts
- Stack: .NET 10 / C# 14, ASP.NET Core minimal APIs, EF Core 10 + PostgreSQL, MassTransit + RabbitMQ
- Solution: `Acme.sln`, 6 projects, CPM enabled (`Directory.Packages.props`)
- Entry executable: `src/Acme.Api` (HTTP :5001), `src/Acme.Worker` (consumers)
- Auth: JWT bearer, policies in `Api/AuthPolicies.cs`
- DB migrations: EF, applied by CI step, NOT on startup

## Where things live
- Endpoints: `src/Acme.Api/Endpoints/<Resource>Endpoints.cs`, registered in `Program.cs:41`
- Business logic: `src/Acme.Application/<Feature>/` — one folder per feature, handler + validator
- Entities: `src/Acme.Domain/`, configured in `src/Acme.Infrastructure/Persistence/Configurations/`

## How to
- Run: `dotnet run --project src/Acme.AppHost` (Aspire — starts DB + broker containers)
- Test: `dotnet test` (integration tests need Docker)
- Add an endpoint: copy pattern in `Endpoints/OrdersEndpoints.cs`; add handler in Application; wire nothing (assembly-scanned)

## Landmines
- `OrderStatus` enum is persisted as int — reordering members corrupts data
- `IClock` must be injected, never `DateTime.UtcNow` (tests freeze time)

## Current state
- Open debt: <count> items, worst: TD-003 (Critical — see TechnicalDebt.md)
- In-flight: <anything evidenced by branches/TODOs, else omit section>
```
